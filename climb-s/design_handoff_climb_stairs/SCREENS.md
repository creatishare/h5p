# 屏幕结构

整页只有一个屏幕，三段式纵向布局：

```
┌──────────────────────────────────────────────────────┐
│  顶栏 TopBar (56px)                                  │
├──────────────────────────────┬───────────────────────┤
│                              │                       │
│  左：题目说明 Problem        │  右：交互操作区        │
│                              │  Playground           │
│  (滚动)                       │  (Scene/Actions/Table)│
│                              │                       │
├──────────────────────────────┴───────────────────────┤
│  底栏 BottomBar (40px)                               │
└──────────────────────────────────────────────────────┘
```

左右两栏的分割比例：`minmax(420px, 1fr) : minmax(560px, 1.2fr)`，中间是 1px 灰色竖分隔线。<= 1100px 时两栏 1:1；<= 820px 时上下堆叠。

---

## 1. 顶栏 TopBar

从左到右：

1. **「体验」+「练习12」tab（激活态）**：左侧带一个青色小胶囊「体验」（背景 `#c4ecff`，文字 `#1a7fc0`，圆角 6px），右侧是 `练习12` 标题；激活态下方有蓝色 3px 底线。
2. **「提示」tab**：右侧带橙红色 `+1` 圆形 badge。
3. **设置图标按钮**：六边形图标，hover 时背景 `#f4f6fa`。
4. **弹性空间** → 推右侧两个按钮到最右。
5. **竖向分隔条**（1×22px）。
6. **「提交」ghost 按钮**：左侧纸飞机 icon。
7. **「选关」ghost 按钮**：左侧三横线 icon。

## 2. 左侧 · 题目说明

可滚动区，内边距 `28px 36px`。区段顺序：

1. **题目描述** — 介绍 n 级台阶、每次走 1 或 2 级。
2. **操作说明** — 4 条 ol 列表，告诉学生右上 1234 切关卡、点 +1/+2 走、记录表自动填、重置可重玩。文中出现的按钮以行内 keycap chip 表示（绿色 `走一级 +1` / 橙色 `走两级 +2`）。
3. **规律观察** — 引导观察「最后一步从 n-1 或 n-2 来」。
4. **完成目标** — 行内 `f(n) = ? + ?` 公式横幅；? 是虚线圆角空白框，等待学生总结。
5. **小贴士** — 列出 n=4 的 5 种走法示例，每个示例用浅灰胶囊高亮。

右上角浮动一个 48×48 的圆形头像（戴眼镜的男生卡通），SVG 实现。

斜体 `<em>` 改成衬线 Times，用来强调数字与变量 `n`。

## 3. 右侧 · 交互操作区 Playground

三段纵向：

### 3.1 Playground tabs (44px)
- 左：`● 交互演示` tab（蓝点 + 文字，激活下划线 2px 蓝色）。
- 右：「学习进度：n/4」+ 140×6px 进度条，蓝色填充。

### 3.2 舞台 Stage Scene（弹性高度，1fr）
- 背景：蓝白渐变天空 `linear-gradient(180deg, #c8e9ff, #d8eeff 55%, #eaf6ff)`。
- 装饰：左上两朵白云、右上太阳（黄色径向 + 22px 黄色光晕）。
- 顶部居中：胶囊形对话气泡（白底 + 蓝边框），显示当前提示语。
- 左上角：1–4 关卡圆形 chip（30px 圆），状态：默认白边、激活蓝实心、完成绿实心。
- 右下角：「本步贡献」浮卡，结构为 `[青色 tag] [路径文本] [黄色加粗数字]`，激活时绿色边框 + 上抬。
- 角色 hero：56×88 的卡通男孩，圆头 + 蓝色身体写「码」字 + 椭圆阴影；用 `--hero-x / --hero-y` 控制位置，跳跃用回弹缓动。
- 台阶 stairs：右下角靠右排布，1–n 级台阶根据当前关卡动态渲染；颜色见 DESIGN_TOKENS。台阶顶部白色衬线数字、底部小标注（0/1/2/3/4 …）。
- 地面：从场景两侧 24px 处一条蓝灰虚线。

### 3.3 操作工具栏 ActionBar
固定栏，`grid 1fr 1fr auto`，间距 12px，内边距 14×20，顶部 1px 灰线、背景 `#fafbfd`。

- **「+1 走一级台阶」按钮**（绿系）：48px 高、圆角 12px；左侧 `+1` 是白底胶囊数字（衬线、800 字重）。
- **「+2 走两级台阶」按钮**（橙系）：同上结构。当当前关卡只有 1 级时，禁用态：底色 `#edf1f5`、字色 `#8995a8`、`data-disabled="true"`。
- **「重置」按钮**：白底灰边按钮，含逆时针箭头 icon。

被点过的走法按钮保持 `done` 类：2.5px 半透明蓝色外框作为「已尝试」标记。

### 3.4 走法记录表 Record
- 标题行：左「走法记录表」（14px/700），右当前提示「正在探索 n = ? 的走法」（小字、灰）。
- 表格：3 列 — `台阶数 n` / `走法 f(n)` / `状态`。
  - 数字单元格使用衬线字体。
  - 当前行底色 `#eef5ff`、文字 `#1f6dec` 加粗。
  - 完成行的 f(n) 列着绿色加粗，状态列显示绿色「已完成」chip。
  - 状态 chip：圆胶囊 22px 高，三种：`todo` 灰 / `now` 蓝 / `done` 绿。

## 4. 底栏 BottomBar
左侧：`题目说明` tab。右侧：`操作记录` tab + 「展开」按钮（带上箭头）。激活 tab 顶部 2px 蓝色短下划线。

## 5. 完成弹窗 Dialog
原生 `<dialog>`：圆角 14px，padding 24×28，标题 20px 加粗，正文行高 1.7，右下角两个按钮 `继续观察`（ghost）+ `下一关`（蓝色 primary）。
