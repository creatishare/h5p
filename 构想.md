# 构想

## 背景

我是一个线上少儿编程教育公司的内容策划，负责策划c++编程课程的一些教学内容的交互设计；现在我希望建立一个由多个agent组成的系统，帮我编写代码，制作出能够通过交互来形象的演示c++代码效果的h5或者网页。



## 例子

例题：爬楼梯

### 代码：

```c++
#include <bits/stdc++.h>
using namespace std;

int c(int n) 
{
    if(n==1) 
    {
        return 1;
    }
    if(n==2) 
    {
        return 2;
    }
    return c(n-1)+c(n-2);
}

int main() 
{
    int n;
    cin>>n;
    cout<<c(n);
    return 0;
}
```

### 题目：

一个人需要爬楼梯，他每次可以爬一层台阶或者爬两级台阶，问他爬n级台阶有几种不同的爬法。

### 演绎效果：

要通点选的操作，让学生理解这道题的解题思路。

- 分别展示1、2、3、4级台阶的情况。让学生模拟走的过程。

- 布局分两大部分：操作区 +记录区

  - 操作区需要展示主角、台阶、两个交互按钮。
  - 记录区记录一共有多少种走法用，用表格展示

  | 台阶数量 /n | 走法/f(n) |
  | ----------- | --------- |
  | 1           |           |
  | 2           |           |
  | 3           |           |
  | 4           |           |

- 从1级台阶的情况开始演示

  - 操作区显示地面和1级台阶，地面下标数字0，台阶上标数字1，角色显示在地面站着
  - 操作区弹出两个按钮（1次走1级，1次走2级）
  - 其中，1次走2级按钮呈现灰色不可按状态，按下后会弹出气泡提示：只有1级台阶
  - 点击1次走1级按钮，角色跳到台阶上，弹出弹窗，显示“有1种走法”同步右侧记录区，f(1)的位置显示1
  - 点击下一关按钮后，进入有2级台阶的界面

- 2级台阶的情况演示

  - 操作区显示地面和两级台阶，同理，地面下标数字0，两层台阶分别标数字1、2，角色显示在地面站着
  - 操作区弹出2个按钮（1次走1级，1次走2级）
  - 点击1次走1级后，演示角色从地面分两步走上2级台阶的过程，并在按钮下标出过程“从0➡️1➡️2，走法+1“
  - 点击1次走2级后，演示角色从地面直接跳上2级台阶的过程，并在按钮下标出过程“从0➡️2，走法+1”
  - 弹出弹窗，显示“有2种走法”同步右侧记录区，f(2)的位置显示2
  - 点击下一关按钮后，进入有3级台阶的界面

- 3级台阶的情况演示

  - 操作区显示地面和三级台阶，同理，地面下标数字0，两层台阶分别标数字1、2、3，先不显示角色
  - 操作区弹出两个按钮（最后走1级，最后走2级）
  - 点击最后走1级后，演示角色出现在2号台阶，一步走上3号台阶的过程，并在按钮下标出“……2➡️3，走法+f(2)"
  - 点击最后走2级后，演示角色出现在1号台阶，然后1步走上3号台阶的过程，并在按钮下标出“……1➡️3，走法+f(1)"
  - 弹出弹窗，显示“有f(2)+f(1)=3种走法“同步右侧记录区，f(3)的位置显示f(2)+f(1)=3
  - 点击下一关按钮后，进入有4级台阶的界面

- 4级台阶的情况演示

  - 操作区显示地面和四级台阶，同理，地面下标数字0，两层台阶分别标数字1、2、3、4，先不显示角色
  - 操作区弹出两个按钮（最后走1级，最后走2级）
  - 点击最后走1级后，演示角色出现在3号台阶，一步走上4号台阶的过程，并在按钮下标出“……3➡️4，走法+f(3)"
  - 点击最后走2级后，演示角色出现在2号台阶，然后1步走上4号台阶的过程，并在按钮下标出“……2➡️4，走法+f(2)"
  - 弹出弹窗，显示“有f(3)+f(2)=5种走法“同步右侧记录区，f(4)的位置显示f(3)+f(2)=5，并在最后显示“恭喜你，发现了规律，f(n)=f(n-1)+f(n-2)"
